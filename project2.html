GPA Influences Explorer- Extended

The project begins with thorough data wrangling, where missing values are managed, and outliers in columns such as "Absences" and "StudyTimeWeekly" are handled. The data is then normalised to ensure consistency across machine learning models.

Using supervised learning techniques, specifically Support Vector Machines (SVM) and Decision Tree Classifiers, this project predicts student performance based on factors such as study time, parental support, and GPA. A comparison of model accuracies highlights the strengths of each model in recognising both high- and low-performing students.

After training and assessing the models, the SVM classifier is applied to new student data to predict each student’s performance class (e.g., "GradeClass").

For the comeptition section, my model was in the top 10% of the cohort

This was the first assignment for a unit at Monash University

(Unit: Intro to Data Science)

Questions Explored

The project aimed to address these analytical and machine learning tasks:

    A1. Data Wrangling (4 marks)
    1. Read the ‘Student_List_A2.csv’ file and list the column names.
    2. Replace numerical grade classifications in the ‘GradeClass’ column with their corresponding letter grades ('A', 'B', 'C', 'D', 'F').
    3. Identify missing values in the dataset and replace them with the median of the relevant columns.
    4. Identify a data quality problem in the ‘Absences’ column and remove rows exhibiting this issue.
    5. Examine the 'GPA' and 'GradeClass' columns for data quality issues, propose solutions, and resolve them.
    A2. Supervised Learning
    1. Explain supervised learning, labelled data, and train-test datasets.
    2. Separate features and labels from the data, ensuring logical predictors are included.
    3. Use `sklearn.model_selection.train_test_split` to split the data for training and testing (80%-20%).
    A3. Classification (training)
    1. Describe the need for normalising data and apply appropriate normalisation using `sklearn.preprocessing`.
    2. Build a Support Vector Machine (SVM) model:
      • Explain SVM and its use of kernels.
      • Write the code to train the model.
    3. Repeat model training using another algorithm (e.g., Decision Tree or Random Forest).
    A4. Classification (prediction)
    1. Use the test dataset to predict ‘GradeClass’ using the SVM and alternate classifier.
    2. Display confusion matrices for both models in a 5x5 format.
    3. Compare model performances and justify the better-performing algorithm.
    A5. Independent Evaluation (Competition)
    1. Predict ‘GradeClass’ for students in ‘Student_List_A2_Submission.csv’ using the best model.
    2. Submit predictions in a CSV file with 162 lines, including a header and predictions for each student.
