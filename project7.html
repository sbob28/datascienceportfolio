Movie Recommender System

The Movie Recommender System helps users find movies similar to the ones they enjoy. It works by analysing movie details such as the cast, crew, genres, and plot. Using text analysis and machine learning, it compares movies and identifies those with the most similarities. When a user enters a movie title, the system quickly searches a database of over 4,800 movies and recommends the top 5 most similar ones. This project makes it easy to discover new films based on personal preferences and can be enhanced with more advanced recommendation techniques in the future.

Key Features

    Data Wrangling & Preprocessing:
    • The system cleans and organises the movie dataset, handling missing values and merging relevant data (e.g., movies and credits datasets).
    • It extracts key information like cast, crew, genres, and plot summaries to create a structured dataset.
    Text-Based Similarity Matching:
    • The extracted movie details are combined into a single descriptive tag for each movie.
    • This tag helps in comparing movies based on their content.
    Vectorisation with Count Vectoriser:
    • The system converts text into numerical data using Count Vectorisation, which creates word frequency representations.
    Global Word Frequency Analysis:
    • It processes a dataset of 4,806 movies and extracts the 9,000 most frequently used words to build a strong comparison model.
    Feature Representation:
    • Each movie is represented as a feature vector in a large multi-dimensional space, making it easier to compare movies mathematically.
    Cosine Similarity Calculation:
    • The system measures the closeness of movies using Cosine Similarity, which determines how similar two movies are based on their text data.
    Recommendation System:
    • When a user enters a movie title, the system finds and displays the top 5 most similar movies based on content analysis.

      Jupyter Notebook

        In Jupyter Notebook, the Movie Recommender System was built by loading and preprocessing data, extracting features, and computing similarities. The dataset was cleaned, transformed, and vectorised using Count Vectorisation. Cosine Similarity was then applied to compare movies, and a function was created to generate recommendations. Visualisations and exploratory data analysis (EDA) helped refine the process.

        There are comments throughout the notebook to better understand the code and the processes which were applied.

        VSCode

          VS Code was used to write, test, and run the Streamlit-based Movie Recommender System. It helped in coding the UI, integrating the recommendation logic, and fetching movie posters using the TMDb API. The environment allowed for smooth debugging, importing necessary libraries, and managing pickle files for storing movie data and similarity scores. Streamlit was then used to deploy the interactive web app.

        Streamlit

          Streamlit was used to build and deploy the interactive web app for the Movie Recommender System. It provided a user-friendly interface with a dropdown menu for movie selection, a button to generate recommendations, and columns to display movie titles and posters. It made the application easy to use without needing complex web development.
