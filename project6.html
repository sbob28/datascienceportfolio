Cycling Through Sleep

Inspiration

The idea for this project arose from the desire to wake up feeling refreshed and energised for the day ahead. Understanding that REM sleep is integral to overall health, we aimed to create a convenient tool to help users incorporate optimal sleep cycles into their daily routines.

What It Does

Cycling Through Sleep calculates the closest REM sleep cycle you can complete based on your desired wake-up time. For example, if itâ€™s 11 PM and you want to wake up at 6 AM, the tool calculates the optimal times to wake up after completing full REM cycles. In this case, it may suggest waking up at 5:30 AM after completing five REM sleep cycles.

Focus on Data Analysis

This project leverages time-series data analysis to calculate optimal wake-up times based on REM sleep cycles, helping users wake up feeling refreshed and less groggy.

  • Sleep cycles last approximately 90 minutes, including REM sleep periods. The tool ensures users wake up at the end of a cycle rather than in the middle, minimizing grogginess.

  • The tool takes two key inputs: current time and desired wake-up time. These inputs form the basis for determining feasible wake-up times that align with complete REM cycles.

  • Using the inputs, the tool calculates multiple wake-up options and suggests the one closest to the user's desired time while ensuring alignment with natural sleep patterns.

How We Built It

JavaScript: We developed the source code in JavaScript to handle sleep cycle calculations.

Bixby Developer Studio: This allowed us to integrate the logic into a voice assistant framework, enabling users to interact with the tool through voice commands.

Challenges We Ran Into

Initially, we planned to use Google Assistant but faced compatibility issues because tools like Dialogflow had been deprecated. Transitioning to Bixby required us to learn new tools and frameworks while ensuring accurate communication between the JavaScript logic and the Bixby platform.
